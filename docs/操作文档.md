## 1. 环境准备

### 1.1 依赖

- Python ≥ 3.12  
- Node.js ≥ 20  
- 可选：Docker、Ollama（本地意图模型）

### 1.2 启动后端

```powershell
cd F:\feishu
copy .env.example .env   # 首次
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev,full]"
playwright install chromium   # 登录态源需要
alembic upgrade head
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 1.3 启动前端

```powershell
cd F:\feishu\frontend
npm install
npm run dev
```

访问：http://127.0.0.1:5173

## 2. 自然语言检索与自动首查

1. 打开 **新建检索**  
2. 输入自然语言，例如：  
   `最近1个月的安徽省区域内的服务器招标信息都有哪些`  
3. 点击 **解析意图**（优先 API → Ollama → 规则）  
4. 在表单中确认/修改关键词、区域、日期、定时  
5. 普通任务点击 **确认并立即检索**；定时任务默认开启 **创建后立即执行首轮**
6. 页面进入真实等待状态，完成后原地展示成功/部分成功/失败、数量、成功与失败来源
7. 下载 Word 报告，或进入采集结果/任务列表继续查看

过期的「今天 9:00」会被拦截，请改为立即执行或有效时间。

大模型配置见 [LLM双模式配置.md](./LLM双模式配置.md)。

---

任务创建成功但首查失败时不会删除任务，可直接点击 **重试执行**。定时任务首查失败时仍保持后续调度，最近一次失败会记录在执行历史中。

## 3. 查看与手工重试

1. 自动首查结束后可直接查看当前页结果；无需再到任务列表点击一次
2. 打开 **任务列表** 可查看中文状态、手工重试、暂停/恢复以及执行记录
3. 每次执行将调用：
   - `ccgp` 中国政府采购网  
   - `cebpub` 中国招标投标公共服务平台  
   - `login_portal` 登录态门户（若已登录）  
4. 在 **采集结果** 按来源、`data_mode` 或交付任务筛选公告
5. 点击公告进入详情抽屉，查看结构化字段、PDF 页码证据、原文、数据质量与 AI 分析
6. 详情操作分为：**浏览器打开官方页**、**导入官方文件**、**自动重新采集**、**仅重新解析**、**重新分析**；人工校正必须填写原因并进入审计记录
7. 官方页空白或详情链路失败时，在常用浏览器下载官方 PDF/HTML 后导入。系统校验公告标题或项目编号，原始文件不落盘；上限为 20 MB、PDF 100 页

注意：ccgp 限速较严，请勿连续高频点击。

## 4. 登录态数据源（必读）

### 4.1 合规原则

| 允许 | 禁止 |
|------|------|
| Playwright **可见**浏览器 | 自动识别/绕过验证码 |
| 用户**手动**输入账号密码 | 代码中保存明文密码 |
| 本地 storage state | 将 state/Cookie 提交 Git |
| 仅免费会员可见内容 | 破解付费墙 |
| 登录失败时公开源继续 | 伪造登录成功数据 |

### 4.2 初始化登录（推荐：数据源界面）

1. 打开前端 **数据源** 页：http://127.0.0.1:5173/sources  
2. 确认页面显示的登录门户配置正确
3. 点击 **启动登录浏览器**  
4. 在弹出的同一招采门户中完成登录
5. 在弹出控制台按 Enter，或等待自动保存  
6. 页面显示「已有登录态文件」后，点 **检查 login_portal**  

对应 API：

- `POST /api/login/start` `{"wait_seconds":600}`
- `POST /api/login/stop`  
- `DELETE /api/login/state`  
- `GET /api/login/status`  

### 4.3 初始化登录（备用脚本）

```powershell
cd F:\feishu
# 双击 scripts\run_login_init.bat
# 或:
cd backend
.\.venv\Scripts\python.exe -m app.tools.login_init --wait 600
```

步骤：

1. 弹出浏览器，打开配置的登录页  
2. **手动登录**（验证码自行完成）  
3. 确认已进入登录后页面  
4. 回到终端 **按 Enter** 或等待自动保存  
5. 状态保存到 `data/browser_states/login_portal_state.json`（已 gitignore）  

### 4.4 Linux / macOS

```bash
chmod +x scripts/login_source.sh
./scripts/login_source.sh
```

### 4.5 配置自定义门户

编辑 `.env`：

```bash
LOGIN_SOURCE_ENABLED=true
LOGIN_SOURCE_HOME_URL=https://your-portal.example/
LOGIN_SOURCE_LOGIN_URL=https://your-portal.example/login
LOGIN_SOURCE_SEARCH_URL=https://your-portal.example/search?q={keyword}
LOGIN_SOURCE_STATE_FILE=login_portal_state.json
LOGIN_SOURCE_LOGIN_MARKERS=请登录,立即登录,会员登录
LOGIN_SOURCE_LOGGED_IN_MARKERS=退出登录,个人中心,我的账号
```

`HOME_URL`、`LOGIN_URL`、`SEARCH_URL` 必须属于同一门户。系统检测到跨站配置时会停用登录源并明确提示；`ccgp`、`cebpub` 等公开源仍会继续执行。登录门户由环境配置统一管理，页面不临时切换到其他站点。

可使用您有权使用的**免费账号**对接商业招采聚合站或企业采购门户。

### 4.6 登录失效

- 接口/前端提示「登录状态失效」  
- 重新运行 `login_source.ps1`  
- **公开源任务仍可正常执行**  

### 4.7 前端查看

- **数据源** 页：查看 `login_portal` 健康状态  
- **设置** 旁也可通过 API `GET /api/login/status` 查看引导步骤（不含 Cookie）

## 5. Word 报告

1. 执行采集成功后自动生成 Word  
2. 命名：`{用户原始问题}_{yyyyMMddHHmm}.docx`  
3. 打开 **报告中心** 下载历史报告  
4. 报告含：封面、查询条件、执行统计、结果列表、字段原文标签、PDF 页码证据、资格条款与 AI 建议
5. 增量场景下结果列表主要为新增/「内容发生变化」条目
6. 如需修复历史报告或补齐详情，进入 **任务列表**，点击 **重新采集未去重完整报告**。该操作把本轮每个来源采到的合格记录分别写入完整快照，可能重复项会保留并标注；后台主记录和交付历史仍按去重规则维护，不会把无变化历史公告重复记为已交付。
7. Word 的“投标决策分析”仅基于公告原文字段：展示时效、资格条款、信息缺口和下一步核验动作，不给出中标概率；模型补充内容必须通过字段证据校验。

详情未获得时，报告写“详情未获取，无法提取”，完整度显示“不可评估”；只有已获得完整正文但确实找不到字段时，才写“原文未明确说明”。“招标人”可规范化为采购主体，但会保留 `source_label=招标人`；交易平台不会作为代理机构，文件售价不会作为预算。

可在 **设置 → 企业画像** 维护产品能力、服务区域、资质及有效期、案例、交付限制、代理/联合体能力。配置后会生成资格逐条匹配矩阵；未配置时仍提供通用分析。

API：

- `GET /api/reports`  
- `GET /api/reports/download/{filename}`

## 7. 定时任务

1. 自然语言中带「每天 9:00 发送」等表达时，确认后创建 `scheduled` 任务  
2. **创建后立即执行首轮** 默认开启；关闭后只等待计划时间
3. 首轮或手工执行后任务仍保持 `scheduled`；单次执行的结果在执行历史中单独显示
4. 任务列表可：**立即执行 / 执行记录 / 暂停 / 恢复 / 删除**
5. 服务重启后会从数据库恢复未暂停的定时任务
6. 单次任务过期不会静默调度；每日/每周/每月到点执行并生成**增量** Word
7. 同一任务执行中禁止并发重复触发（HTTP 409）

## 8. 常见问题

**Q: 解析没有关键词/区域？**  
A: 在确认页手工补充后确认。

**Q: 采集结果为空？**  
A: 可能被二次过滤、站点限流或关键词过窄；查看执行返回的 `sources_failed`。

**Q: 8000 端口已有程序时会发生什么？**
A: 当前版本 FusionBid 会直接打开；当前工作区旧版本会询问是否安全重启；未知进程或无法确认归属的 FusionBid 只提示 PID 并退出，不会强制结束。

**Q: Playwright 安装失败？**  
A: 使用 `python -m playwright install chromium`，检查网络代理。

**Q: 能否自动登录？**  
A: 赛题要求手动登录；本系统**不实现**自动填密与验证码绕过。
