# FusionBid 开源调研与技术选型

> 版本：v1.0  
> 日期：2026-07-17  
> 目的：在继续开发前固化「能否直接复用 / 仅可参考 / 必须自研」的结论，并划定合规边界。  
> 项目：FusionBid 智标聚合助手（2026 AI 先锋未来人才大赛——超聚变企业命题）

---

## 1. 调研结论（摘要）

| 结论 | 说明 |
|------|------|
| **无整仓可替换方案** | 未发现同时满足「自然语言意图解析 + 人工确认 + 公开源 + 登录态源 + 清洗去重增量 + Word 报告 + 定时任务」的开源项目。 |
| **公开采集可参考** | [TenderCrawler](https://github.com/ir-st/TenderCrawler) 对 **cebpub / ccgp** 的站点拆分、限速与快照对比最有价值。 |
| **产品形态可参考、实现不可照搬** | [BidMonitor-AI](https://github.com/zhiqianzheng/BidMonitor-AI) 具备多站监控与 AI 过滤，但存在「绕过反爬」类表述，与赛题合规要求冲突。 |
| **意图解析必须自研** | 无「招投标 NL → 区域/时间/频率/调度」的可直接安装完整库；LLM 层可借鉴 Instructor / Structured Outputs。 |
| **主体继续在本仓库自建** | 在现有 FastAPI + React 骨架上分阶段交付，参考外部项目的**思路与站点模式**，不 fork 替换主仓。 |

**决策**：先固化本文档；后续按阶段二起继续自研，阶段三公开源优先参考 TenderCrawler 的适配器思路。

## 2. 调研范围与方法

### 2.1 检索关键词（中英文）

- 招投标 / 政府采购 / 公共资源 / 招标公告 采集、爬虫、聚合、监控  
- tender / procurement / bidding scraper、ccgp、cebpub  
- 自然语言意图解析、LLM structured output、Instructor  
- 登录态 Playwright storage state（合规方案对照）

### 2.2 评估维度

1. 是否覆盖赛题硬性验收项  
2. 是否可直接作为 FusionBid 依赖或子模块  
3. 合规性：登录、验证码、反爬、付费内容、密钥管理  
4. 技术栈与本仓库一致性（Python 3.12+、FastAPI、异步、Playwright）  
5. 可维护性与页面结构变更风险  

### 2.3 不纳入「直接可用」的情形

- 仅脚本/课程作业级爬虫，无统一适配器与错误处理  
- 明确以绕过反爬、绕过登录为卖点的实现  
- 仅商业 API/MCP 查询、无法证明「真实网站采集」链路  
- 与赛题技术栈严重不符且改造成本高于自研

### 3.1 TenderCrawler（优先参考 · 公开数据源）

| 项 | 内容 |
|----|------|
| 地址 | https://github.com/ir-st/TenderCrawler |
| 许可 | MIT（以仓库 LICENSE 为准） |
| 技术 | Python 3.10+、Flask、requests、BeautifulSoup |
| 站点 | 中国招标投标公共服务平台（cebpub）、中国政府采购网（ccgp） |
| 可借鉴 | ① `sites/registry` 式站点注册扩展  
|  | ② cebpub JSON API 列表检索  
|  | ③ ccgp HTML 列表解析与详情字段  
|  | ④ 请求间隔（ccgp 建议 ≥4s）  
|  | ⑤ 快照对比（新增/消失）用于增量思路  
|  | ⑥ 列表与详情分离、招标时间字段解析 |
| 缺口 | 无 NL 意图、无登录态、无 Word、无 LLM 摘要、非 FastAPI 异步架构 |
| 用法 | **阶段三**实现 `TenderSourceAdapter` 时参考逻辑并在本仓库重写；不整仓依赖、不复制违规绕过逻辑 |

**与 FusionBid 映射建议**

```text
TenderCrawler sites/*  →  backend/app/sources/ccgp_*.py / cebpub_*.py
TenderCrawler compare  →  backend/app/deduplication/ + DeliveryHistory 增量
TenderCrawler delay    →  core 配置 CRAWL_RATE_LIMIT / 每源超时重试
```

### 3.5 意图解析与 LLM 结构化输出

| 方案 | 地址/来源 | 评价 |
|------|-----------|------|
| **Instructor** | https://github.com/567-labs/instructor · https://python.useinstructor.com/ | 将 LLM 输出落到 Pydantic，支持多厂商 OpenAI 兼容接口；**适合阶段二 LLM 层** |
| OpenAI Structured Outputs / tools | 官方 SDK 能力 | 与 `LLM_BASE_URL` 兼容网关配合时优先；无 Key 时必须规则降级 |
| 通用意图识别文档（如云厂商 NLU 方案） | 各类帮助中心 | 仅方法论；无招投标槽位 |
| 招投标专用 NL 意图开源库 | — | **未发现可直接安装即用** |

**阶段二选型（已实现）**

```text
优先级：兼容 API → 本地 Ollama → 规则降级
实现：httpx + OpenAI 兼容 /v1/chat/completions + JSON 约束
Ollama：list / pull / 选用模型；支持自定义模型名与 Base URL
运行时：data/llm_runtime.json（不含 API Key）
必选：严格校验 + 前端人工确认/修改
文档：docs/LLM双模式配置.md
禁止：对五个官方验收案例硬编码答案
```

## 4. 与赛题硬性要求的差距矩阵

| 赛题硬要求 | 开源覆盖情况 | FusionBid 策略 |
|------------|--------------|----------------|
| NL 意图：关键词/区域/时间/执行时间/频率 | 基本空白 | **自研** parsers + llm |
| LLM + 规则降级 + 校验 + 人工确认 | 无完整链路 | **自研** |
| ≥2 真实互联网来源 | 多项目仅公开源 | 公开源参考 TenderCrawler；登录源自研 |
| 至少一个登录后可见有效信息 | 合格合规实现稀缺 | Playwright 可见浏览器 + 用户手动登录 + storage state |
| 禁止绕过验证码/登录/反爬/付费墙 | 部分项目冲突 | **明确禁止**引入相关代码 |
| 清洗、二次过滤、跨源去重 | 部分仅 URL/ID 去重 | 自研 cleaners + deduplication |
| 增量推送（DeliveryHistory） | TenderCrawler 快照对比可参考 | 自研，失败执行不标记已推送 |
| Word 报告与命名规范 | 未见同款 | 阶段六 python-docx 自研 |
| 定时 once/daily/weekly/monthly | 有简易 cron，缺完整任务模型 | APScheduler + SearchTask 持久化 |
| data_mode live/fixture 标注 | 少见 | 模型已预留，演示必须标注 |

## 5. 合规边界（强制）

以下规则对所有「参考开源」与自研代码同时生效：

1. **不得**绕过验证码、登录安全机制、反爬策略或访问控制。  
2. **不得**破解付费会员或获取免费权限之外的信息。  
3. **不得**在代码/仓库中保存明文账号密码、API Key、Cookie、storage state。  
4. 登录态数据源：Playwright **可见**浏览器 → 用户**手动**登录 → 本地 storage state（`.gitignore`）→ 失效时提示重新登录。  
5. 不自动处理验证码；日志禁止输出 Token/Cookie/密码。  
6. 登录源失败时，公开源仍应继续执行。  
7. 请求须限速、超时、重试与并发上限；测试避免对真实站点高频访问（mock/fixture/录制响应）。  
8. 引用开源思路时保留出处说明；许可证以原仓库为准（MIT 等允许参考重写时，仍建议在文档中标注）。  
9. **禁止**把「绕过反爬」类实现以任何形式并入本仓库。  
10. 无附件不得伪造；摘要不得编造预算/截止时间/联系人。

## 6. FusionBid 技术选型（固化）

### 6.1 已选定（阶段一）

| 层级 | 选型 |
|------|------|
| 后端 | Python ≥3.12、FastAPI、Pydantic 2、SQLAlchemy 2、Alembic、SQLite |
| 前端 | React、TypeScript、Vite、Ant Design、React Router、TanStack Query |
| 工程 | Docker Compose、`.env`、Windows/Linux 启动脚本 |

### 6.2 分阶段选型

| 阶段 | 能力 | 选型 | 开源参考 |
|------|------|------|----------|
| 二 | 意图解析 | 自研 rule + LLM（OpenAI 兼容）+ 校验 + 确认 API/页 | Instructor（可选） |
| 三 | 公开源 | httpx + BS4/lxml；统一 `TenderSourceAdapter` | **TenderCrawler**（cebpub/ccgp） |
| 四 | 登录源 | Playwright storage state；失败不阻塞公开源 | 无合格整仓方案 → 自研 |
| 五 | 去重增量 | 多特征去重 + DeliveryHistory | TenderCrawler compare 思路 |
| 六 | Word | python-docx；命名 `{问题}_{yyyyMMddHHmm}.docx` | — |
| 七 | 调度 | APScheduler + 任务持久化 | 通用调度模式，业务自研 |
| 八 | 联调 | 真实双源 + 登录 + 报告 + 文档 | — |

### 6.3 公开数据源候选（阶段三确认）

| 优先级 | 来源 | 类型 | 备注 |
|--------|------|------|------|
| P0 | 中国政府采购网 search.ccgp.gov.cn | 公开 HTML | TenderCrawler 有实践；注意限速与结构变更 |
| P0 | 中国招标投标公共服务平台 cebpub | 公开 JSON/页面 | 同上 |
| P1 | 全国公共资源交易平台 / 省级交易中心 | 公开 | 结构差异大，按演示区域选型 |
| P1 | 省级政府采购网（与查询区域一致） | 公开 | 如安徽/上海等与案例区域对齐 |

登录态候选（阶段四确认，须用户自有合法账号）：

- 商业招投标聚合站的**免费会员可见**列表/详情  
- 或企业采购门户需登录后可见的公告区  

最终选型以「可演示、合规、结构稳定」为准，写入 `docs/数据源说明.md`。

### 6.4 明确不采用

| 项 | 原因 |
|----|------|
| 整仓替换为 BidMonitor-AI / GBS_BidScope | 缺口大 + 合规/架构不符 |
| Selenium「绕过反爬」脚本 | 违反赛题 |
| 仅本地 JSON 冒充 live | 违反赛题 |
| 硬编码五个验收案例答案 | 违反赛题 |
| 未授权账号 / 明文密码入库 | 违反赛题与安全规范 |

## 7. 推荐实施路径（与开发计划对齐）

```text
[已完成] 阶段一：项目骨架
[本文档] 开源调研与技术选型固化
[下一步] 阶段二：意图解析（自研 LLM + 规则 + 校验 + 确认）
        阶段三：公开源（参考 TenderCrawler，本仓适配器实现）
        阶段四：登录态（Playwright 合规）
        阶段五～八：去重增量、Word、调度、联调
```

**引用原则**

- 允许：阅读公开 README/代码，理解站点 URL、字段含义、限速经验，在本仓库用自有架构重写。  
- 禁止：原样拷贝「绕过安全机制」的实现；禁止把第三方密钥或登录态提交进 Git。  
- 文档：阶段三接入具体站点后，在 `docs/数据源说明.md` 写明来源、是否登录、限速与失效处理。

## 8. 风险清单

| 风险 | 影响 | 缓解 |
|------|------|------|
| 政府/公共站点改版 | 解析失败 | 适配器隔离；健康检查；结构异常提示；fixture 回归 |
| 限流 / IP 封禁 | 演示中断 | 严格限速、退避、降低并发；演示预热 |
| 登录态过期 | 登录源不可用 | 明确提示重登；公开源继续 |
| LLM 不稳定/无 Key | 意图解析降级 | 规则解析 + 人工确认必达 |
| 开源项目停更或许可证变更 | 参考失效 | 不把运行时强依赖对方仓库 |
| 合规误用第三方「绕过」代码 | 验收失败/法律风险 | 代码审查 + 本文档第 5 节强制约束 |
