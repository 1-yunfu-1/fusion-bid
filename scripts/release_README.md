# FusionBid 智标聚合助手 — 使用说明（发布包）

> AI 招投标信息聚合：自然语言查询 → 多源采集 → 清洗去重 → Word 报告 → 定时增量  
> 本包为**可运行发布包**，**不含 API Key**，请自行在页面中配置。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 系统 | Windows 10 / 11 |
| Python | **3.12 或更高**（安装时务必勾选 **Add python.exe to PATH**） |
| 网络 | 首次启动需联网安装依赖；采集招投标网站也需能访问外网 |
| 可选 | 本机 Chrome / Edge（登录态采集更稳）；Ollama（本地大模型） |

**本包不需要安装 Node.js。** 前端已预构建，双击启动即可。

---

## 二、一键启动（推荐）

1. 将本文件夹解压到任意路径（路径尽量不要特别长）
2. 双击 **`start.bat`** 或 **`一键启动.bat`**
3. 首次运行会自动：
   - 创建 Python 虚拟环境  
   - 安装后端依赖（可能需数分钟，请耐心等待）  
   - 尝试安装 Playwright 浏览器内核（登录采集用）
4. 看到提示后，用浏览器打开：

### 访问地址

| 用途 | 地址 |
|------|------|
| **系统界面（请打开这个）** | **http://127.0.0.1:8000/** |
| API 文档（开发调试） | http://127.0.0.1:8000/docs |
| 健康检查 | http://127.0.0.1:8000/api/health |

> **注意：**  
> - 正确界面是「系统概览 / 新建检索 / 任务列表…」等网页，**不是**一段 JSON。  
> - 若打开后只看到 `{"name":"FusionBid..."}` 之类内容，请确认是否双击了本包的 `start.bat`，并强制刷新浏览器（Ctrl+F5）。  
> - 关闭黑色启动窗口即停止服务。

## 三、配置 API Key（包内未预置）

本发布包**故意不包含任何 API Key**，请任选一种方式配置：

### 方式 A：页面配置（推荐）

1. 打开 http://127.0.0.1:8000/  
2. 进入 **设置** → **API 配置组**  
3. 填写名称、Base URL、API Key、默认模型后保存  
4. 可保存多组，随时切换

密钥保存在本机 `data/llm_secrets.json`，**请勿把该文件再打包发给别人**。

### 方式 B：环境变量

1. 首次启动会从 `.env.example` 自动生成 `.env`  
2. 用记事本打开 `.env`，填写：

```text
LLM_API_KEY=你的密钥
LLM_BASE_URL=https://你的兼容接口/v1
LLM_MODEL=模型名
```

3. 保存后**重启** `start.bat`

也可用本地 Ollama（在设置页启用，并先在本机 `ollama pull` 模型）。

## 四、基本使用流程

1. **设置**：配置 API Key / 模型（或 Ollama）  
2. **新建检索**：输入自然语言，例如「最近1个月北京市服务器招标」→ 解析 → 确认创建任务  
3. **任务列表**：可 **编辑**、**立即执行**、定时暂停/恢复  
4. **数据源**：公开源 + 登录态源；登录请用页内「启动登录浏览器」  
5. **公告 / 报告**：查看结果，下载 Word 报告（`data/reports/`）

更细的操作见 `docs/操作文档.md`、`docs/Demo演示脚本.md`。

## 六、常见问题

**Q：浏览器打开是 JSON 不是界面？**  
A：请用本包 `start.bat` 启动，打开 **http://127.0.0.1:8000/**，并 Ctrl+F5 刷新。不要只开旧进程或未带 `frontend/dist` 的后端。

**Q：提示找不到 Python？**  
A：安装 Python 3.12+，勾选 Add to PATH，重开终端后再双击 `start.bat`。

**Q：首次启动很慢 / pip 失败？**  
A：需要能访问 PyPI（或配置国内镜像）。网络稳定后重新运行 `start.bat` 即可。

**Q：没有大模型能不能用？**  
A：意图解析可降级到规则模式；效果较差。建议配置 API 或 Ollama。公开源采集不依赖大模型。

**Q：登录网站后仍提示未登录？**  
A：普通浏览器登录无效。请在「数据源」页使用「启动登录浏览器」，在弹出的窗口中登录并保存。

**Q：端口 8000 被占用？**  
A：关闭占用 8000 的其他程序，或修改启动命令中的端口（需同步改访问地址）。
