## 1. 开场（1 分钟）

**口述要点：**

1. 产品名：**FusionBid 智标聚合助手**  
2. 用户用**自然语言**描述需求 → 系统解析意图 → 多源详情采集 → 证据校验 → **Word 决策报告**
3. 数据源：**≥2 公开源**（ccgp、cebpub）+ **1 登录态源**（Playwright 手动登录）  
4. **无硬编码招标结果**；所有结果标注 `data_mode`

打开首页，展示健康检查 `ok`、阶段信息。

## 2. 示例一：立即执行（3 分钟）

**输入（新建检索）：**

```text
最近1个月的安徽省区域内的服务器招标信息都有哪些
```

**演示步骤：**

1. 点击「解析意图」  
2. 指出识别结果：关键词「服务器」、区域「安徽省」、最近 1 个月起止日期、**立即执行**、无定时  
3. 必要时手工微调后点击「确认并立即检索」
4. 页面立即进入真实等待状态（提醒：ccgp 限速，可能需数十秒），不需要跳到任务列表再点一次
5. 在当前页展示成功/部分成功/失败、数量及失败来源，并下载 Word
6. 打开「采集结果」详情抽屉：展示来源、`live` 标注、招标人/代理机构/资格条款、PDF 页码证据与原文
7. 展示 AI 决策枚举、证据 ID、时间倒排、风险与资格矩阵；再打开 Word 核对同一证据链

**话术：** 报告文件名保留用户原问题 + 时间戳，不覆盖历史文件；详情无法核验时会明确显示“仅元数据”，不会把门户通用页当公告正文。

## 3. 示例二：指定月份（2 分钟）

**输入：**

```text
2026年3月份的上海区域内的充电桩招标信息都有哪些
```

**要点：**

- 日期范围 **2026-03-01～2026-03-31**  
- 区域 **上海市**，主题 **充电桩**  
- 立即执行  

若时间紧，可在解析后说明其确认按钮会沿用同一条自动首查路径。

## 4. 示例三：每日定时 + 增量（3 分钟）

**输入：**

```text
最近3个月的上海区域内的充电桩招标信息都有哪些，请汇总后每天9:00发送给我
```

**要点：**

- 定时：`daily` `09:00`；确认页的「创建后立即执行首轮」默认开启
- 确认后自动完成首轮，任务状态仍为 `scheduled`，下次执行时间保留
- 展示任务页中文状态、「执行记录」以及「暂停 / 恢复」
- 再点击一次「立即执行」说明增量：已交付内容不重复进入报告，仅新增/内容变化；点击「重新采集未去重完整报告」则演示本轮每个来源记录都保留的完整快照，可能重复项明确标注，但不重复写交付历史

**话术：** 失败执行不会把未交付内容标成已推送。

## 5. 数据源与登录态（2 分钟）

1. 打开「数据源」：ccgp、cebpub、login_portal

## 7. 收尾与风险说明（1 分钟）

| 点 | 说明 |
|----|------|
| 合规 | 限速、手动登录、不破解付费 |
| 限制 | 网站改版/限流可能导致当次抓取变少 |
| 代码 | 适配器分文件、可测、可 Docker Compose |
| 测试 | 展示 `docs/测试报告.md` 最新结果；外部站使用 mock/fixture，不高频访问 |

## 8. 备用话术（网络不佳时）

若真实站点不可达：

1. 运行 `scripts/e2e_smoke.py` 展示解析与 Word 生成  
2. 打开 `docs/测试报告.md` 展示 mock 覆盖的链路  
3. 展示当次离线冒烟生成的报告；历史报告仅在来源和 `data_mode` 可核实时使用

**切勿**用硬编码假招标列表冒充 live 采集成功。
