# FusionBid 报名材料项目说明

> 用于「2026 AI 先锋未来人才大赛——超聚变企业命题」报名表、作品简介和答辩材料统一表述。以下内容不包含账号、Cookie、API Key 或未经验证的采集数量。

## 项目名称

FusionBid 智标聚合助手

## 一句话介绍

面向企业招采信息发现与投标决策的 AI 助手：用户确认检索条件后自动完成多源首查、可验证详情抽取、资格分析和 Word 决策报告，并按计划持续交付增量。

## 项目简介

FusionBid 将传统的“逐站搜索、手工筛选、复制整理、凭经验判断”整合为可确认、可追溯的闭环。系统从自然语言中识别关键词、区域、日期范围和执行频率；用户确认后立即执行首轮检索。详情正文先做规则快速提取，关键字段存在缺口时再交给 AI，并以公告原文、PDF 页码和证据片段强校验；模型不可用或证据不成立时保留规则结果。最终报告给出资格逐条匹配、时间倒排、风险、材料缺口和下一步建议。

## 核心创新点

1. **确认即执行的完整闭环**：输入、解析、人工确认、自动首查、结果解释和报告下载在同一流程中完成。
2. **可信降级**：在线模型或 Ollama 不可用时切换规则解析；登录源不可用时公开源继续，不用静态结果冒充实时采集。
3. **跨源去重与增量交付**：保留来源及合并依据，只在报告成功交付后写入 DeliveryHistory，失败执行不会误标为已推送。
4. **合规登录态**：Playwright 可见浏览器、用户手工登录、本地 storage state；不绕过验证码、反爬或付费访问控制。
5. **结果可追溯**：界面标注来源和 `data_mode=live/fixture`；无附件保持空数组，无法核验的详情明确标记，不编造预算、截止时间或联系人。
6. **证据约束的 AI 决策**：每个字段和建议引用证据 ID；结论限定为四档，不生成无依据的中标概率；可选企业画像形成资格匹配矩阵。
7. **原始完整快照**：重新采集时不做跨源去重且不读写增量交付历史，每个来源记录均进入报告；安全上限触发时明确标记截断。

## 技术方案

- 后端：Python 3.12、FastAPI、Pydantic 2、SQLAlchemy 2、Alembic、SQLite
- 前端：React、TypeScript、Vite、Ant Design、TanStack Query
- 采集与报告：HTTP/HTML 适配器、PDF.js 文本层与本地 OCR 降级、Playwright 手工登录态、证据抽取、跨源去重、python-docx
- 调度：单次、每日、每周、每月；支持暂停、恢复、手工重试和执行历史
- 数据源：`ccgp`、`cebpub` 公开源，以及配置一致时启用的 `login_portal`

## 当前边界

- 不引入 Celery/Redis、邮件或 IM 推送；“发送”在当前版本表示生成可下载的增量报告。
- 真实结果数量受站点可达性、页面改版和访问限制影响；不得用 fixture 冒充 live。
- 登录门户需要参赛者自有合法账号和现场手工登录；未登录时仍可演示公开源链路。
- 当前版本：`v1.0.0`；运行阶段标识：`phase8-integration`；默认时区：`Asia/Shanghai`。
