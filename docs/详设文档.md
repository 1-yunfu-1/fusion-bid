### 3.1 核心业务流程（目标态）

```text
自然语言输入
→ 意图解析（LLM + 规则降级 + 人工确认）
→ 立即执行 / 建立定时任务
→ 多数据源检索（公开 + 登录态，登录失败不阻塞公开源）
→ 列表页 / 详情页 / 附件
→ 清洗 → 条件过滤 → 跨源去重 → 增量识别
→ 核心内容摘要（不编造）
→ Word 报告 → 执行记录 → 页面下载
```

## 4. 数据库模型

### 4.1 TenderAnnouncement（招投标公告）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID/str PK | 主键 |
| title | str | 标题 |
| source_name | str | 来源网站名 |
| source_url | str | 可回跳原始链接 |
| source_item_id | str? | 来源侧 ID |
| requires_login | bool | 是否来自登录态源 |
| data_mode | enum | `live` / `fixture` |
| publish_time | datetime? | 发布时间 |
| region / province / city | str? | 地区 |
| keywords | JSON | 关联关键词 |
| summary | text? | 核心内容摘要 |
| clean_content | text? | 清洗后正文 |
| raw_content | text? | 原始正文 |
| attachment_links | JSON | 附件 URL 列表，无则 `[]` |
| crawl_time | datetime | 抓取时间 |
| content_hash | str? | 正文哈希 |
| deduplication_key | str? | 去重键 |
| first_seen_at / last_seen_at | datetime | 首次/最近发现 |
| created_at / updated_at | datetime | 审计字段 |

### 4.2 SearchTask（检索/订阅任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID/str PK | 主键 |
| original_query | text | 用户原始问题 |
| parsed_intent | JSON | 结构化意图 |
| keywords / regions | JSON | 关键词、区域 |
| start_date / end_date | date? | 时间范围 |
| execute_immediately | bool | 创建后是否执行首轮（普通任务固定为 true，定时任务默认 true） |
| schedule_enabled | bool | 是否定时 |
| schedule_type | str? | once/daily/weekly/monthly |
| execute_time | str? | 如 `09:00` |
| timezone | str | 默认 Asia/Shanghai |
| status | str | draft/confirmed/running/scheduled/paused/done/failed/expired/deleted |
| created_at / updated_at | datetime | 审计 |

### 4.3 TaskExecution（单次执行）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID/str PK | 主键 |
| task_id | FK | 关联任务 |
| started_at / finished_at | datetime | 起止 |
| status | str | pending/running/success/failed/partial |
| trigger_type | str | initial/manual/scheduled |
| sources_requested / sources_succeeded | JSON | 请求/成功来源 |
| raw_result_count | int | 原始条数 |
| filtered_result_count | int | 过滤后 |
| duplicate_count | int | 重复数 |
| incremental_count | int | 本次新增 |
| report_path | str? | 后端内部 Word 路径；API 仅返回文件名与下载地址 |
| error_message | text? | 错误信息 |

### 4.4 DeliveryHistory（增量交付）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID/str PK | 主键 |
| task_id | FK | 任务 |
| announcement_id | FK | 公告 |
| content_hash | str | 交付时哈希 |
| first_delivered_at / last_delivered_at | datetime | 首次/最近交付 |
| report_id | str? | 关联报告/执行 |

**增量原则**：仅成功写入报告后标记已交付；内容显著变化可再次输出并标注「内容发生变化」。

### 4.5 扩展表（骨架预留，后续阶段启用）

- `SourceDuplicateLink`：主记录与被合并来源、去重依据、URL 列表
- `SourceStatus`：数据源启用状态、登录态探测结果（亦可运行时聚合）

## 5. API 规划

基础前缀：`/api`  
响应：JSON；错误统一 `{ "detail": "..." }`

### 5.1 基础接口（已实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查（应用、数据库、时区、版本） |
| GET | `/api/meta` | 系统元信息（名称、阶段、时区） |

### 5.2 业务接口（已实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/parse` | 自然语言意图解析 |
| POST | `/api/parse/confirm` | 人工确认/修改意图并创建任务 |
| GET/POST | `/api/tasks` | 任务 CRUD |
| POST | `/api/tasks/{id}/execute` | 执行；支持 initial/manual、quick/standard/complete、强制重抽取；调度器内部使用 scheduled |
| GET | `/api/tasks/{id}/executions` | 执行历史、触发类型与报告下载地址 |
| GET | `/api/sources` | 数据源列表与状态 |
| POST | `/api/sources/{name}/health` | 单源健康检查 |
| POST | `/api/sources/{name}/login-init` | 触发登录初始化说明/状态 |
| GET | `/api/announcements` | 公告列表；支持来源、data_mode、任务和生命周期筛选 |
| GET | `/api/reports` | 报告列表 |
| GET | `/api/reports/download/{filename}` | Word 下载 |
| POST | `/api/tasks/{id}/pause` 等 | 暂停/恢复/删除 |

## 6. 数据源适配器接口（已实现）

```python
class TenderSourceAdapter:
    source_name: str
    requires_login: bool

    async def health_check(self) -> HealthResult: ...
    async def search(self, query: SearchQuery) -> list[ListItem]: ...
    async def fetch_detail(self, item: ListItem) -> DetailResult: ...
    async def extract_attachments(self, detail: DetailResult) -> list[str]: ...
```

约束：

- 每源独立文件，禁止混写
- 支持启用/禁用、超时、限速、重试退避、并发上限
- 登录态：Playwright 手动登录 + storage state（`.gitignore`）
- 登录源失败时公开源继续

**当前来源**：

1. 公开：中国政府采购网（ccgp）、中国招标投标公共服务平台（cebpub）
2. 登录：统一配置的招采门户（login_portal；合规手动登录，不绕过验证码）

## 7. 意图解析设计

1. **LLM 结构化解析**（`LLM_*` 环境变量，Key 不入库）  
2. **规则降级**（区域词典、相对时间、频率关键词、dateparser）  
3. **严格校验**（缺区域/关键词、过期「今天 9:00」、冲突日期 → 提示修正）  
4. **人工确认页**  

禁止对验收案例硬编码答案。

---

## 8. 清洗、去重、报告、调度（概要）

详见赛题：二次条件过滤、跨源去重保留官方优先、摘要不编造、Word 命名  
`{原始问题}_{yyyyMMddHHmm}.docx`、定时任务仅推送增量。

`parsers/`、`sources/`、`cleaners/`、`deduplication/`、`reports/`、`scheduler/` 与 `llm/` 均已接入主流程。

## 9. 安全与合规

- API Key / 账号密码不写代码、不进 Git  
- browser storage state 入 `.gitignore`  
- 不绕过验证码/登录/反爬；不破解付费内容  
- 日志禁止输出 Cookie/Token/密码  
- 请求限速与合理重试

## 11. 开源调研结论（摘要）

详见 [开源调研与技术选型.md](./开源调研与技术选型.md)。

- **无**可直接替换本仓库的完整开源方案。
- 公开数据源实现优先**参考** [TenderCrawler](https://github.com/ir-st/TenderCrawler)（cebpub / ccgp、站点注册、限速、快照对比）。
- [BidMonitor-AI](https://github.com/zhiqianzheng/BidMonitor-AI) 仅作产品对照；禁止引入「绕过反爬」类实现。
- 意图解析、登录态 Playwright、Word 报告、DeliveryHistory 增量等**必须自研**。
- LLM 结构化输出可借鉴 Instructor / OpenAI 兼容 Structured Outputs，业务 schema 自建。
